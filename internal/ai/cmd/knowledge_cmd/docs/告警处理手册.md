# 暴力破解攻击告警
告警解释：检测到针对SSH/RDP/Web登录页面的暴力破解攻击，短时间内同一源IP发起大量认证请求。
威胁等级：P1（高危）
MITRE ATT&CK：T1110 - Brute Force
处置流程：
1. 调用 query_firewall_logs 查询攻击源IP的全部流量记录，确认攻击范围
2. 在 TopPolicy 防火墙中添加临时封禁规则，阻断攻击源IP（封禁时长24小时）
3. 检查目标系统认证日志，确认是否存在成功登录记录
4. 如发现成功登录，立即执行账户锁定并启动横向移动排查
5. 通知安全管理员审核封禁策略，评估是否需要永久封禁

# 端口扫描告警
告警解释：检测到来自外网IP的大规模端口扫描行为，可能为攻击前的侦察阶段。
威胁等级：P2（中危）
MITRE ATT&CK：T1046 - Network Service Discovery
处置流程：
1. 调用 query_firewall_logs 按源IP过滤，统计扫描端口范围和频率
2. 确认扫描源是否为已知安全扫描器（如公司内部安全审计工具）
3. 若为未知来源，在 TopPolicy 中配置 deny 规则封禁该IP段
4. 检查被扫描端口对应的服务是否存在已知漏洞
5. 输出扫描分析报告，包含扫描时间线、目标端口TOP10、命中规则列表

# DDoS流量告警
告警解释：检测到异常大流量涌入，可能为分布式拒绝服务攻击，目标服务可用性受到威胁。
威胁等级：P0（紧急）
MITRE ATT&CK：T1498 - Network Denial of Service
处置流程：
1. 立即调用 query_security_alerts 确认告警详情和影响范围
2. 调用 query_firewall_logs 分析流量特征（源IP分布、协议类型、目标端口）
3. 启用 TopIDP 的 DDoS 防护策略，配置流量清洗规则
4. 联系网络团队确认上游运营商是否已触发黑洞路由
5. 持续监控业务可用性，每5分钟检查一次服务状态
6. 攻击缓解后输出事件报告，记录攻击峰值流量、持续时间、缓解措施

# 恶意软件通信告警
告警解释：防火墙检测到内网主机与已知恶意C2服务器的通信流量，可能存在主机失陷。
威胁等级：P0（紧急）
MITRE ATT&CK：T1071 - Application Layer Protocol（C2通信）
处置流程：
1. 调用 query_firewall_logs 查询涉事内网IP的全部外联流量
2. 确认C2通信的目标IP/域名是否在威胁情报中标记
3. 立即在 TopPolicy 中阻断该内网主机的所有外联通信
4. 通知终端安全团队对该主机执行全盘查杀
5. 排查该主机是否存在横向移动行为（检查同网段其他主机的异常连接）
6. 收集 IOC 指标（C2 IP、通信端口、恶意进程哈希），更新防护规则

# Web应用攻击告警（SQL注入/XSS）
告警解释：TopWAF 检测到针对Web应用的SQL注入或XSS攻击尝试。
威胁等级：P2（中危）
MITRE ATT&CK：T1190 - Exploit Public-Facing Application
处置流程：
1. 查看 WAF 日志确认攻击载荷内容和目标URL
2. 调用 query_firewall_logs 查询攻击源IP的历史访问记录
3. 评估攻击是否成功（检查HTTP响应码和响应体）
4. 如攻击成功，立即通知开发团队进行漏洞修复
5. 在 TopWAF 中添加针对性拦截规则
6. 对攻击源IP实施临时封禁（7天）

# 未授权访问告警
告警解释：检测到内网用户访问未授权的敏感资源或系统，可能存在权限滥用或账户被盗。
威胁等级：P1（高危）
MITRE ATT&CK：T1078 - Valid Accounts
处置流程：
1. 调用 query_security_events 查询该用户的近期操作记录
2. 调用 query_firewall_logs 分析该用户IP的访问模式
3. 联系该用户确认操作是否本人发起
4. 如确认非本人操作，立即锁定账户并重置凭据
5. 检查该账户近期的权限变更记录
6. 排查是否存在其他被盗账户

# 防火墙策略违规告警
告警解释：检测到流量命中了应当被拒绝的防火墙策略，可能存在配置异常或策略绕过尝试。
威胁等级：P3（低危）
处置流程：
1. 调用 query_firewall_logs 查询命中的规则ID和流量详情
2. 在 TopPolicy 管理控制台确认规则配置是否正确
3. 检查是否存在策略冲突（优先级较高的 allow 规则覆盖了 deny 规则）
4. 如发现配置异常，修正策略并记录变更原因
5. 定期审计防火墙策略，确保最小权限原则

# 安全事件严重等级定义
- P0（紧急）：影响核心业务可用性或存在数据泄露风险，需15分钟内响应
- P1（高危）：存在明确的攻击行为或账户安全威胁，需30分钟内响应
- P2（中危）：检测到攻击尝试但未成功，需2小时内响应
- P3（低危）：策略合规性问题或低风险异常，需24小时内响应
